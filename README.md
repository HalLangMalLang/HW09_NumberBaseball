## 🎮 게임 로직 흐름

### 1️⃣ 게임 시작
- 플레이어들이 로비에서 준비 완료
- 게임 시작 시 공통된 고유한 3자리 숫자(1~9, 중복 없음) 생성
- 턴 순서 무작위 배정 및 첫 번째 플레이어부터 턴 시작

### 2️⃣ 턴 관리
- 현재 턴 플레이어에게 제한 시간(30초) 부여
- 타이머는 서버에서 관리하고 모든 클라이언트에 동기화
- 턴 종료 시 다음 플레이어로 자동 전환
- 시간 초과 시 자동으로 다음 턴으로 넘어감

### 3️⃣ 입력 검증
- **1단계 (IsMinigameInput)**: 채팅이 게임 입력인지 판별
  - 3자리 숫자
  - 1~9 범위
  - 0 제외
- **2단계 (CheckNumberMatch)**: 게임 규칙 검증
  - 중복 숫자 검사
  - Strike/Ball 계산 및 결과 반환

### 4️⃣ 게임 진행
- 플레이어가 공통 정답 숫자를 추측
- Strike: 숫자와 위치 모두 일치
- Ball: 숫자는 있지만 위치 다름
- OUT: 일치하는 숫자 없음
- 3 Strike 달성 시 해당 플레이어 승리

### 5️⃣ 승리 조건
- 3 Strike를 먼저 달성한 플레이어 승리
- 게임 종료 후 결과 화면 표시

### 6️⃣ 퇴장 처리
- 플레이어 퇴장 시 현재 턴 인덱스 기준으로 턴 순서 재조정
- 남은 플레이어가 1명이면 즉시 승리 처리
- 타이머 자동 정리 및 다음 턴 시작

### 7️⃣ 네트워크 동기화
- 서버에서 게임 상태 관리 (GameState)
- 턴 정보, 타이머, 플레이어 상태를 클라이언트에 복제
- 모든 게임 로직은 서버에서 검증 후 클라이언트에 전달
